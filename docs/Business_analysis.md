# Бизнес-анализ проекта 3D-сегментации глиом с визуализацией неопределённости

Планируется разработать систему поддержки принятия врачебных решений для автоматической 3D-сегментации глиом высокой и низкой степени злокачественности у взрослых на мультимодальных МРТ-снимках. В результате пользователь получает МРТ-снимки с выделенными ключевыми областями: вся опухоль (**whole tumor, WT**), ядро опухоли (**tumor core, TC**) и накапливающая контраст опухоль (**enhancing tumor, ET**), а также возможность наложить карту неопределённости, чтобы визуализировать области, где модель недостаточно уверенна в своей сегментации. 

### Бизнес-цель проекта (Business Understanding)

*   **Сократить время, затрачиваемое врачом на ручную разметку** 

    -> снижение трудозатрат на анализ одного случая 

    -> увеличение пропускной способности отделения

    -> потенциальное сокращение Time-to-Treatment для пациента 

*   **Повысить объективность и воспроизводимость сегментации и оценки объёма опухоли**
     
    -> снижение межэкспертной вариабельности

    -> обеспечение более надёжного динамического наблюдения

    -> поддержка планирования лечения и мультидисциплинарного обсуждения

    -> снижение числа спорных/неоднозначных случаев, требующих повторного экспертного пересмотра

### Существующие решения

1.  **Ручная сегментация**
    *   Подвержена субъективизму.
    *   Большие затраты времени квалифицированного персонала.

2.  **Коммерческие PACS-системы**
    *   Всегда дают уверенный ответ даже при ошибке.
    *   Врачу не доступна внутренняя логика, по которой происходит сегментация.
    *   Дорогие.
    *   Могут требовать специфической настройки под конкретные протоколы МРТ.

### Организационная структура

**Заказчик:**
*   Диагностические центры.
*   Государственные больницы, частные клиники.
*   Возможно фармкомпании и контрактно-исследовательские организации (объективизировать динамику изменения опухоли в ответ на экспериментальное лечение в клинических исследованиях).

**Основной пользователь:**
*   Нейрохирурги.
*   Радиотерапевты (радиационные онкологи).
*   Клинические онкологи.
*   Медицинские исследователи, занимающиеся оценкой эффективности противоопухолевой терапии.
*   Врачи лучевой диагностики.

### Контакты и рабочие чаты

*   В рамках учебного проекта созданы локальные каналы коммуникации для взаимодействия с экспертами и обсуждения технических решений.

Дополнительно можно использовать данные из открытых датасетов **BraTS (MICCAI)**, **BraTS-Africa** и **TCGA-GBM** как внешний датасет для проверки переносимости модели. ### Риски проекта и план по их уменьшению

1.  **Сроки**
    
    Длительное время обучения 3D-сетей может привести к задержкам на этапе моделирования.
    *   *План:* Использование предобученных весов (Transfer Learning) и проверенных архитектур (например, nnU-Net или MONAI 3D U-Net).


2.  **Доменный сдвиг (Domain Shift)**
    
    Разные датасеты могут иметь отличия сканеров, протоколов, качества МРТ.
       *   *План:* Применение техник нормализации интенсивности и стандартизации пространственного разрешения (resampling).


3.  **Возможный перекос данных в сторону определенных подтипов опухолей**
    *   *План:* Использование продвинутой аугментации данных (вращение, масштабирование, эластичные деформации) и стратифицированной кросс-валидации.

## 1.2 Решаемые задачи с точки зрения аналитики (Data Mining goals)

### Техническая постановка задачи
Задача медицинской 3D-семантической сегментации. На вход подаются четыре МРТ-модальности: **T1, T1ce, T2, FLAIR**. На выходе модель должна построить маски опухолевых областей: **WT, TC, ET**.

### Метрики
Для оценки качества сегментации будут использоваться стандартные в медицине метрики:
*   **Dice coefficient** для WT, TC, ET (оценка степени перекрытия предсказанной маски и эталона).
*   **Hausdorff Distance 95th percentile** для WT, TC, ET (оценка точности определения границ опухоли в мм).
*   **Sensitivity / Recall**.
*   **Precision**.

### Критерии успешности
*   **Минимальный порог:** Средний Dice > 0.75 для WT.
*   **Оптимальный уровень:** Dice >= 0.85 для WT, >= 0.75 для TC и >= 0.70 для ET.

Дополнительно результат можно оценивать визуально: наложение предсказанной маски на МРТ-срезы, сравнение с экспертной разметкой и расчёт объёма опухоли.

## 1.3 План проекта (Project Plan)

1.  **Бизнес-анализ (Business Understanding):** Определение целей, оценка рисков, постановка технических задач.
2.  **Анализ данных (Data Understanding):** Исследовательский анализ (EDA) медицинских снимков, проверка модальностей, визуализация срезов и масок.
3.  **Подготовка данных (Data Preparation):** Преобразование форматов, ресэмплинг к изотропному разрешению, нормализация интенсивности, создание загрузчиков данных.
4.  **Моделирование (Modeling):** Выбор и реализация нейросетевой архитектуры, настройка функции потерь (Dice Loss + CrossEntropy), обучение.
5.  **Оценка результата (Evaluation):** Валидация на тестовых данных, расчет метрик Dice и Hausdorff, анализ визуальных несоответствий.
6.  **Внедрение (Deployment):** Разработка демонстрационного интерфейса (Streamlit или веб-сервис) для загрузки МРТ-файлов и отображения сегментированных зон.
